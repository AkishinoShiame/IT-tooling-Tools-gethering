# BIOSはBasic Input Output Systemの略

主にはキーボード、マウスなどの基本入力ハードウェアの管理。
実際にはマザーボードにとあるチップです、小さいなシステム・コードが入ってる場所。
現在には色んな機能と仕込むが入って、グラフィックスなどのコードも入れてます。

### BIOSとOSの関係

機械が起動している時、BIOSに設定されによって、ハードウェアの読み方法が変更できる。
例えば以下の設定が弄り出来ます。
- LagacyやUEFIモード： OSのディスク分割によってMBRやGPTと言う違いがあります。
- Security Boot： オンやオフ。UEFIモード限定、OSのブート途中にヴィールス防止の為に。
- セキュリティーキー： 上の機能やBitLockerが使う機能、キーがなくしたら、BitLockerが自動解錠出来ない場合が出る。
- デイスクの読み方： IDEはちょっと古きの、大体AHCIが今のメイン。他にはRAIDも有ります。

### BIOSの更新

BIOSは非常に重要ですけど、普段は大体弄らない方が良いかも。
だがしかし、たまにOSの更新や、ハードウェアの更新について、支援度が低いになる。
あとは中に実行のもコードので、たまに弱点が出る場合もあり、その時は更新が必要です。

更新方法は、大体公式サイトから、最新のをダウンロードし、Windows OSで実行で良いだが、
たまにOSが起動できずの場合があって、その時は別のパソコンから、USBドライブをfat32にフォマートしBIOSファイルを中にコッピーする、
それからBIOSの中に、firmware updateとかを開いて、USB挿して更新できます。

古手の場合はRufusを使って、FreeDOSの簡単システムも入れられます。

### UEFI x GPT と　Lagacy x MBR

BIOSは二つモードがあります。
旧世代にはハードウェアが今の強さがないから、Lagacyの時はMBRの記録方です。
- 一つのディスクは主なpartitionが４個しか出来ず、サブはもっと多いですが、OSには入れず。
- 最大支援ディスクサイスは2Tしか記録でき。
新世帯のUEFIには、上に述べたを改善のため、GPTの記録方法が作れた。
- ここにはOSのブートファイルを分離し、EFI(Fat32)のpartitionしか読める。
- EFIに入れるレコードによって、別のpartitionをどれがOSの認識し、ブートプロセスを続けます。
- ディスクの制限や分割制限もいっぱいに改善された。
